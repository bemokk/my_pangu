# Wipha Case Plot Script Split Design

## Goal

Refactor `Buoy/plots/plot_wipha_case_analysis.py` so that each Wipha case output can be generated by an independent script while keeping a one-command orchestrator.

## Scope

Do not change analysis methods, selected platforms, output file names, statistics, or plot appearance. Only reorganize code.

## Design

Create `Buoy/plots/wipha_case_common.py` for shared configuration, data loading, aggregation, typhoon track loading, forecast center extraction, statistics, and reusable utilities.

Create four focused plotting scripts:

- `Buoy/plots/plot_wipha_track_buoy_locations.py`
- `Buoy/plots/plot_wipha_buoy_wind_timeseries.py`
- `Buoy/plots/plot_wipha_buoy_wind_statistics_table.py`
- `Buoy/plots/plot_wipha_track_forecast_error.py`

Keep `Buoy/plots/plot_wipha_case_analysis.py` as a thin orchestrator that calls the four scripts' `make_figure`/`main` style entry points.

## Compatibility

Existing helper tests should import from `wipha_case_common.py`. Add a lightweight test ensuring the four scripts are importable.

## Out of Scope

Do not modify unrelated files currently present in the worktree, especially `trash/get_typhoon_path.py` and `typhoon_2506_Wipha.csv`.
