# 下一阶段实验方案：融合未来目标时效风场、当前波浪场与残差预测的中国近海波浪场多时效预报模型

## 1. 前期工作基础

前一阶段已完成基于 ERA5 风浪数据的序列到序列波浪预测实验。实验以西北太平洋区域过去 24 小时逐小时 10 m 风场作为输入，包括 `u10` 和 `v10` 两个变量，预测中国近海未来 `+6h`、`+12h`、`+24h`、`+48h` 和 `+72h` 五个预报时效下的有效波高、平均波周期和平均波向。

模型采用 ConvLSTM 编码器提取过去 24 小时风场的时空演变特征，并通过多个预测头分别输出不同未来时效的波浪场。波向变量采用 `cos(MWD)` 和 `sin(MWD)` 表示，以避免 0° 与 360° 之间的角度不连续问题。实验已完成数据读取、样本构造、归一化、训练、验证、测试和持续性基线对比等完整流程。

前期实验结果表明，仅使用过去 24 小时风场，模型已经能够在 `+24h`、`+48h` 和 `+72h` 时效上优于持续性基线，说明历史风场中确实包含对未来中长期波浪变化有用的信息，模型能够学习风场演变与未来波浪场之间的统计映射关系。

## 2. 前期实验结果与主要问题

前期实验的总体测试损失低于持续性基线，说明模型整体上具备一定预测能力。然而，分时效结果显示，模型在短时效预测中仍存在明显不足。

具体表现为：

1. 在 `+24h`、`+48h` 和 `+72h` 时效上，模型对有效波高、平均波周期和平均波向的预测均优于持续性基线，说明该模型对中长期波浪变化具有一定预测能力。
2. 在 `+6h` 和 `+12h` 时效上，模型表现不如持续性基线，尤其是 `+6h` 预测误差明显偏大。
3. 训练过程中出现过拟合现象。训练损失持续下降，但验证损失在第 2 轮后逐渐升高，说明模型对训练集的拟合能力继续增强，但泛化能力没有同步提升。

造成上述问题的主要原因包括：

首先，当前模型只输入过去 24 小时风场，而没有输入初始化时刻 `t0` 的实际波浪场。对于 `+6h` 和 `+12h` 等短时效预测而言，当前海况本身是最重要的信息之一。持续性基线直接使用 `t0` 时刻波浪场作为未来预测，因此在短时效上具有天然优势，而当前模型需要仅根据历史风场间接推断当前海况，导致短时效预测困难。

其次，当前模型没有显式输入未来风场。波浪的生成、发展和传播受到未来风场持续作用影响，尤其在 `+24h`、`+48h` 和 `+72h` 等较长时效下，未来风场对波浪变化具有重要驱动作用。仅使用过去风场时，模型只能从历史风场中推断未来天气系统演变趋势，信息不够充分。

最后，当前模型直接预测未来波浪场的绝对值，没有充分利用“未来波浪场通常是在当前海况基础上发生变化”这一特征。对于短时效预测而言，直接预测完整未来波浪场难度较高，而预测相对于当前波浪场的变化量更符合物理过程，也更有利于模型学习。

## 3. 下一阶段改进方向

针对前期实验暴露出的问题，下一阶段拟开展“风场 + 当前波浪场 + 残差预测”改进实验。该实验选择第二组输入方案，即在过去 24 小时风场基础上，加入 ERA5 未来目标时效风场，包括 `+6h`、`+12h`、`+24h`、`+48h` 和 `+72h` 风场。同时，将 `t0` 时刻的实际波浪场作为初始海况输入，并将模型输出形式由直接预测未来波浪场改为预测未来波浪场相对于当前波浪场的变化量。

改进后的基本思想为：

```text
未来波浪场 = 当前波浪场 + 模型预测的波浪变化量
```

即：

```text
Wave(t0 + lead) = Wave(t0) + ΔWave(t0 + lead)
```

其中，`Wave(t0)` 表示初始化时刻的当前波浪场，`ΔWave(t0 + lead)` 表示模型预测的未来波浪变化量，`lead` 分别对应 `+6h`、`+12h`、`+24h`、`+48h` 和 `+72h`。

该方案的核心改进包括三点：

1. 加入 `t0` 时刻波浪场，为模型提供当前海况初始条件，重点改善短时效预测。
2. 加入 ERA5 未来目标时效风场，为模型提供未来风场驱动信息，重点提升中长期时效预测。
3. 采用残差预测方式，使模型学习未来波浪相对于当前海况的变化，而不是从零开始预测完整波浪场。

## 4. 输入与输出设计

### 4.1 历史风场输入

历史风场输入仍采用过去 24 小时逐小时 ERA5 10 m 风场：

```text
t0 - 23h, t0 - 22h, ..., t0
```

输入变量包括：

```text
u10, v10
```

张量形式为：

```text
past_wind: [B, 24, 2, H_in, W_in]
```

其中，`B` 为批量大小，`24` 为历史时间步数，`2` 代表 `u10` 和 `v10` 两个风场变量，`H_in` 和 `W_in` 分别为输入风场区域的纬向和经向网格数。

### 4.2 未来目标时效风场输入

下一阶段实验采用 ERA5 未来目标时效风场作为理想未来风场输入。选取与波浪预测目标一致的 5 个未来时效：

```text
t0 + 6h, t0 + 12h, t0 + 24h, t0 + 48h, t0 + 72h
```

输入变量包括：

```text
u10, v10
```

张量形式为：

```text
future_wind: [B, 5, 2, H_in, W_in]
```

其中，`5` 表示 5 个未来目标时效，`2` 表示 `u10` 和 `v10` 两个风场变量。

本阶段使用 ERA5 未来风场的目的不是直接代表业务预报能力，而是构建“理想未来风场驱动”下的上限实验，用于判断加入未来风场信息后，波浪预测性能理论上能够提升到什么程度。后续可进一步将 ERA5 未来风场替换为盘古气象模型预测风场，评估实际预报条件下的性能变化。

### 4.3 当前波浪场输入

为了提供初始化海况，模型新增 `t0` 时刻波浪场输入，包括：

```text
SWH, MWP, cos(MWD), sin(MWD)
```

张量形式为：

```text
wave0: [B, 4, H_out, W_out]
```

其中，`4` 表示有效波高、平均波周期、波向余弦和波向正弦四个变量，`H_out` 和 `W_out` 表示中国近海输出区域的空间网格数。

### 4.4 模型输出

模型不再直接输出未来波浪场绝对值，而是输出未来各时效相对于 `t0` 当前波浪场的变化量：

```text
delta_wave: [B, 5, 4, H_out, W_out]
```

其中，5 个时效分别为：

```text
+6h, +12h, +24h, +48h, +72h
```

最终预测结果通过残差形式得到：

```text
wave_pred = wave0 + delta_wave
```

也就是说，对于每一个预测时效：

```text
wave_pred(+6h)  = wave0 + delta_wave(+6h)
wave_pred(+12h) = wave0 + delta_wave(+12h)
wave_pred(+24h) = wave0 + delta_wave(+24h)
wave_pred(+48h) = wave0 + delta_wave(+48h)
wave_pred(+72h) = wave0 + delta_wave(+72h)
```

## 5. 模型结构设计

下一阶段模型建议采用三分支特征提取与多时效预测头结构。

### 5.1 历史风场编码分支

历史风场分支继续使用 ConvLSTM 编码器，用于提取过去 24 小时风场的时间演变和空间结构特征：

```text
past_wind → ConvLSTM Encoder → past_feature
```

该分支主要提供前期风浪发展背景，例如强风区位置、风场移动趋势、台风或季风系统的演变信息。

### 5.2 未来风场编码分支

未来风场分支用于编码 ERA5 未来目标时效风场。由于未来风场只选取 `+6h`、`+12h`、`+24h`、`+48h` 和 `+72h` 五个离散时效，可采用两种实现方式：

第一种方式是共享 CNN 编码器。对每个未来时效风场分别使用同一个 CNN 编码器提取特征，得到：

```text
future_feature(+6h)
future_feature(+12h)
future_feature(+24h)
future_feature(+48h)
future_feature(+72h)
```

第二种方式是使用轻量级时序编码器，对五个未来时效风场进行整体编码，提取未来风场驱动特征。

考虑到本阶段是改进验证实验，建议优先采用第一种方式，即共享 CNN 编码器。该方式结构简单，便于与多时效预测头对应。

### 5.3 当前波浪场编码分支

当前波浪场分支采用 CNN 编码器，对 `t0` 时刻的初始波浪场进行特征提取：

```text
wave0 → CNN Encoder → wave0_feature
```

该分支为模型提供当前海况信息，重点改善短时效预测。

### 5.4 多时效残差预测头

在预测阶段，将历史风场特征、当前波浪场特征和对应未来时效风场特征进行融合，并送入对应时效的预测头。

例如：

```text
[past_feature, wave0_feature, future_feature(+6h)]  → Head_6h  → delta_wave(+6h)
[past_feature, wave0_feature, future_feature(+12h)] → Head_12h → delta_wave(+12h)
[past_feature, wave0_feature, future_feature(+24h)] → Head_24h → delta_wave(+24h)
[past_feature, wave0_feature, future_feature(+48h)] → Head_48h → delta_wave(+48h)
[past_feature, wave0_feature, future_feature(+72h)] → Head_72h → delta_wave(+72h)
```

每个预测头输出 4 个变量的变化量：

```text
ΔSWH, ΔMWP, Δcos(MWD), Δsin(MWD)
```

最终通过残差连接得到未来波浪场预测值。

## 6. 损失函数设计

模型训练时仍以未来 ERA5 波浪场作为监督标签。虽然模型内部输出的是变化量，但损失函数建议作用在最终还原后的未来波浪场上，即：

```text
wave_pred = wave0 + delta_wave
loss = Loss(wave_pred, wave_true)
```

基础损失函数可采用 MSE：

```text
Loss = mean((wave_pred - wave_true)^2)
```

为进一步改善短时效预测，可在基础实验后尝试加入不同时效权重。例如：

```text
Loss =
1.5 × Loss_6h
+ 1.2 × Loss_12h
+ 1.0 × Loss_24h
+ 1.0 × Loss_48h
+ 1.0 × Loss_72h
```

但在第一轮实验中，建议先采用等权重损失，以便判断“当前波浪场 + 未来目标时效风场 + 残差预测”本身带来的改进效果。若短时效仍不足，再进一步引入时效加权损失。

## 7. 对照实验设计

为了明确各类输入信息的贡献，建议设置以下对照实验：

| 模型编号              | 输入信息                                | 输出方式      | 实验目的            |
| ----------------- | ----------------------------------- | --------- | --------------- |
| M1                | 过去 24 小时风场                          | 直接预测未来波浪场 | 前期基准模型          |
| M2-direct         | 过去 24 小时风场 + ERA5 未来目标时效风场          | 直接预测未来波浪场 | 检验未来目标时效风场的贡献   |
| M2-wave0-direct   | 过去 24 小时风场 + ERA5 未来目标时效风场 + t0 波浪场 | 直接预测未来波浪场 | 检验当前海况输入的贡献     |
| M2-wave0-residual | 过去 24 小时风场 + ERA5 未来目标时效风场 + t0 波浪场 | 残差预测      | 推荐改进模型，检验残差学习效果 |

其中，`M2-wave0-residual` 是本阶段重点实验模型。

通过上述对照，可以分别回答以下问题：

1. 加入 ERA5 未来目标时效风场后，中长期波浪预测是否进一步提升？
2. 加入 `t0` 当前波浪场后，`+6h` 和 `+12h` 短时效预测是否明显改善？
3. 残差预测是否比直接预测未来波浪场更稳定？
4. 新模型是否能够在保持 `+24h`、`+48h` 和 `+72h` 优势的同时，提高短时效预测能力？

## 8. 评价指标

模型评价仍采用逐时效、逐变量评价方式，并与持续性基线进行对比。

建议评价指标包括：

1. 有效波高 SWH：RMSE、MAE、Bias、Correlation。
2. 平均波周期 MWP：RMSE、MAE、Bias、Correlation。
3. 平均波向 MWD：角度 MAE 或角度 RMSE。
4. 总体归一化测试损失。
5. 相对于持续性基线的提升率。

其中，相对提升率可表示为：

```text
Improvement = (Baseline Error - Model Error) / Baseline Error × 100%
```

若提升率为正，说明模型优于持续性基线；若提升率为负，说明模型不如持续性基线。

## 9. 预期结果

本阶段改进模型预期具有以下效果：

1. `+6h` 和 `+12h` 预测性能显著改善。由于模型加入了 `t0` 当前波浪场，并采用残差预测方式，模型可以直接在当前海况基础上学习未来变化，而不是从历史风场中间接推断当前海况。
2. `+24h`、`+48h` 和 `+72h` 预测性能进一步提升。由于模型加入了 ERA5 未来目标时效风场，能够更直接地利用未来风场驱动信息，减少仅依赖过去风场推断未来变化的不确定性。
3. 模型训练可能更加稳定。残差预测降低了输出学习难度，模型只需学习未来相对于当前的变化量，有助于改善短时效预测和整体泛化能力。
4. 新模型可作为后续使用盘古气象模型预测风场驱动波浪预测的过渡方案。ERA5 未来风场实验可视为理想未来风场条件下的上限实验，后续可将未来风场输入替换为盘古预测风场，评估实际应用条件下的性能损失。

## 10. 成功判据

本阶段实验可采用以下标准判断是否成功：

1. `+6h` 和 `+12h` 的 SWH、MWP 和 MWD 指标明显接近或优于持续性基线。
2. `+24h`、`+48h` 和 `+72h` 继续保持对持续性基线的优势。
3. 总体测试损失低于前期基准模型。
4. 验证损失不再在训练早期快速恶化，过拟合现象有所缓解。
5. `M2-wave0-residual` 优于 `M2-direct` 和 `M2-wave0-direct`，说明当前波浪场输入和残差预测均具有实际贡献。

## 11. 后续扩展方向

如果本阶段 ERA5 未来目标时效风场实验取得明显改进，后续可进一步开展以下实验：

1. 将 ERA5 未来目标时效风场替换为盘古气象模型预测风场，评估真实风场预报误差对波浪预测的影响。
2. 将未来风场输入由 5 个目标时效扩展为 6 小时间隔序列，例如 `+6h`、`+12h`、`+18h`、……、`+72h`，以更完整地表征未来风场持续作用。
3. 引入早停、weight decay 和 dropout 等正则化方法，进一步缓解过拟合。
4. 在低分辨率实验验证有效后，将空间步长由 16 逐步提高到 8 或 4，以提升波浪场空间细节表达能力。
5. 扩展多年 ERA5 数据，提高模型对季风、台风和不同季节海况的泛化能力。

## 12. 阶段目标总结

下一阶段实验的核心目标是：在前期“过去 24 小时风场预测未来波浪场”模型基础上，引入 ERA5 未来目标时效风场、`t0` 当前波浪场和残差预测机制，构建更加符合物理过程的多时效波浪预测模型。

该模型将当前海况作为初始条件，将未来目标时效风场作为外部驱动，将模型学习目标从“直接预测未来波浪场”转变为“预测未来波浪相对于当前海况的变化量”。通过该设计，预期能够重点改善 `+6h` 和 `+12h` 短时效预测，同时进一步提升 `+24h`、`+48h` 和 `+72h` 中长期预测效果，为后续引入盘古气象模型预测风场、构建实际可用的风浪场预报系统奠定基础。
