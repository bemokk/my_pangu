# 六组风浪场预测实验结果总结（2016-2025）

更新时间：2026-06-21

## 1. 实验目标

本阶段实验目标是比较不同输入信息对未来多时效波浪场预测的影响。

统一任务：

- 输入区域：5°N-45°N，95°E-150°E
- 输出区域：15°N-40°N，105°E-135°E
- 历史输入长度：过去 24 小时逐小时风场
- 预测时效：+6h、+12h、+24h、+48h、+72h
- 预测变量：SWH、MWP、cos(MWD)、sin(MWD)
- 训练数据：2016-2025 年 ERA5，Zarr 格式
- 主要训练精度：bf16

需要特别说明：当前六组正式结果里，残差模型还没有完整 20 轮结果。已完成或可比较的六组主要是 direct/no residual 方案，其中一组是第 19 轮后中断的 partial run。

## 5. 主要结论

1. 未来风场非常关键。

   M1 只使用过去 24h 风场，最佳 val_loss 为 0.258384。加入未来目标时效风场后，M2 的最佳 val_loss 降到 0.146669，改善约 43.2%。这说明波浪场预报不仅依赖历史风，还强烈依赖未来风强迫。

2. t0 波浪场有明显帮助。

   在同样使用未来目标时效风场的情况下，加入 t0 波浪场后，E4 的最佳 val_loss 从 E2 的 0.146669 降到 0.112539，进一步改善约 23.3%。这说明当前波浪状态本身是很强的初始条件。

3. 未来 72h 连续风场 + t0 波浪场是目前最强方案。

   E5 的最佳 val_loss 为 0.082638，是当前六组里最优结果。相对 M1 改善约 68.0%，相对 E4 也继续改善约 26.6%。

   从物理指标看，E5 在平均 SWH RMSE、MWP RMSE 和 MWD MAE 上都是最优：

   - 平均 SWH RMSE：0.2189
   - 平均 MWP RMSE：0.4654
   - 平均 MWD MAE：10.03°
   - +72h SWH RMSE：0.2480
   - +72h MWD MAE：11.48°

4. continuous72 输入效果好，但训练代价明显更高。

   E5 平均训练速度约 209.8 samples/s，而 E4 约 1429.5 samples/s。也就是说，使用未来 72h 连续风场后，训练速度大约只有目标时效风场方案的 15% 左右。

   这是因为模型需要处理 72 个未来小时的风场序列，而不是只处理 5 个目标 lead time 的风场。

5. stride=4 当前没有带来预期收益。

   E6 使用更高空间分辨率的 stride=4，但最佳 val_loss 为 0.136846，反而弱于 stride=8 的 E4。这个结果不能简单理解为“高分辨率无用”，因为 E6 同时改变了 batch size、训练轮数和空间采样密度，loss 的统计分布也会变。

   更合理的解释是：当前模型容量、batch size 或学习率设置还没有充分适配 stride=4。直接提高空间分辨率会增加优化难度和训练成本。

6. 残差方案还没有完成正式对比。

   当前六组正式结果均为 direct/no residual。之前的 residual 实验只跑到第 3 轮后已清理到隔离区，因此不能用于严肃对比。

## 6. 当前最佳模型

当前推荐作为主线模型：

`M2wave0c72__in_past24hWind+future72hWind+waveT0__direct_nores__s8_b1024_h64_bf16__ep20_pause40__y2016-2025`

对应方案：

- 输入：过去 24h 风场 + 未来 72h 连续风场 + t0 波浪场
- 输出：未来 +6/+12/+24/+48/+72h 波浪场
- 方式：direct/no residual
- stride：8
- batch size：1024
- hidden channels：64
- precision：bf16
- epoch：20
- 最佳 val_loss：0.082638

## 7. 下一步建议

1. 优先补齐 residual 消融实验。

   需要完整跑完：

   - M2-wave0-residual + futureLeadWind，stride=8，20 epoch
   - M2-wave0-residual + future72hWind，stride=8，20 epoch

   这样才能回答“残差预测是否优于直接预测”。

2. 给训练脚本补完整断点续训。

   目前 checkpoint 主要保存模型权重和配置，没有保存 optimizer state、scheduler state、epoch 编号。建议补充完整 checkpoint，避免长时间训练中断后只能做 weights-only 续训。

3. 对 continuous72 模型做速度优化。

   E5 效果最好，但速度明显慢。可以考虑：

   - 缓存未来风场编码特征
   - 简化 future wind encoder
   - 将 72h 连续风场压缩为较低频率输入，例如每 3 小时或每 6 小时采样
   - 对未来风场做多尺度编码，而不是完整逐小时 ConvLSTM

4. stride=4 需要单独设计实验。

   不建议直接把 stride 从 8 改到 4 就期待指标提升。更稳妥的做法是：

   - 固定模型结构，单独调 batch size 和 learning rate
   - 对 stride=4 使用更长训练或更强正则
   - 比较同一验证网格上的物理指标，而不只看训练 loss

5. 暂不建议直接上 stride=1 大规模训练。

   之前 stride=1 实验出现过系统级重启，且显存、内存、功耗压力都很高。正式尝试前应先解决硬件稳定性或降低负载。
